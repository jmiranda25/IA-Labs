Context: Stack from step 1. Auth = Clerk. Storage = Replit Object Storage.
Everything inside Replit + Agent 4.

Goals:

DB (Drizzle):
- resources: id, title, slug (unique), type (enum: 'link' | 'file' |
  'course'), url (text nullable, for link/course), file_path (text
  nullable, the Replit Object Storage path for type='file'),
  description (markdown), cover_url (text nullable),
  author_id (text fk → users.id), published (bool default false),
  created_at
- resource_tags: id, resource_id (fk on delete cascade), tag (text).
  Unique (resource_id, tag). Indexed on tag.

Backend (Express + Orval):
- GET    /api/resources?q=&tags=&type=&cursor=
         → only published unless requester is the author or admin
- GET    /api/resources/:slug
- POST   /api/resources (any authenticated user; published=false by default)
- POST   /api/resources/:slug/file (multipart, type='file' only;
         uploads to `resources` bucket in Replit Object Storage;
         ACL: public-read for published rows, owner+admin-read otherwise)
- POST   /api/admin/resources/:slug/publish   → admin only, sets published=true,
         updates ACL to public-read, calls notify() to the author
- POST   /api/admin/resources/:slug/reject    → admin only, deletes the row
         and the file (if any), calls notify() to the author with reason
- DELETE /api/resources/:slug                 → author or admin

Frontend (artifacts/react-vite):
- /recursos: filterable grid. shadcn search input, type tabs (Todos /
  Enlaces / Archivos / Cursos), tag chips below the search bar
  (multi-select; URL-synced query params).
- /recursos/[slug]: cover, title, description (markdown), author chip,
  tag chips, CTA "Abrir" (link/course) or "Descargar" (file → goes
  through GET /api/storage/objects/* with ACL check).
- /recursos/nuevo: submission form (RHF + Zod). After submit, show
  "Tu recurso queda en revisión hasta que un admin lo apruebe".
- /admin/recursos: pending queue with Aprobar / Rechazar; preview pane
  shows the resource exactly as members would see it once published.

Deliverables: end-to-end resource flow with moderation.
