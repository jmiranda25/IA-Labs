Context: Stack from step 1. Auth = Clerk. Drizzle schema lives at lib/db/.
Real-time = SSE (wired in step 13). Cover images = Replit Object Storage.
Everything inside Replit + Agent 4.

Goals:

DB (Drizzle schema + migration via `pnpm --filter @workspace/db run generate`):
- events: id (uuid pk), title, slug (unique), description (text, markdown),
  starts_at (timestamptz), ends_at (timestamptz), location (text nullable),
  capacity (int nullable), is_online (bool default false),
  meeting_url (text nullable), cover_url (text nullable),
  created_by (text fk → users.id), created_at (default now)
- rsvps: id (uuid pk), event_id (fk on delete cascade),
  user_id (text fk → users.id on delete cascade),
  status (enum: 'going' | 'interested' | 'cancelled'),
  created_at, updated_at. Unique (event_id, user_id).
- Indexes: events(starts_at), rsvps(event_id, status), rsvps(user_id).

Backend (Express + Orval):
- Auth-required reads:
    GET /api/events?status=upcoming|past&mode=online|in_person&q=…&cursor=…
    GET /api/events/:slug   → event + counts { going, interested } + my rsvp
- Authenticated writes:
    POST /api/events/:slug/rsvp { status }
- Admin only (requireAdmin):
    POST   /api/admin/events           → create
    PATCH  /api/admin/events/:slug     → update
    DELETE /api/admin/events/:slug
    POST   /api/admin/events/:slug/cover  → multipart, `events` bucket in
                                            Replit Object Storage, ACL
                                            public-read, admin-write.
- Slugify title server-side; ensure uniqueness with a numeric retry suffix.
- Capacity logic: when capacity is reached, /api/events/:slug/rsvp with
  status='going' returns 409 'event_full'.

Frontend (artifacts/react-vite):
- /eventos: shadcn cards in a grid, sorted by starts_at asc when upcoming,
  desc when past. Tabs: Próximos / Pasados. Filters: Online / Presencial /
  Todos. Search by title.
- /eventos/[slug]: cover hero, title, datetime range (Intl.DateTimeFormat
  with the user's locale), location or "Online", meeting_url shown only to
  RSVP'd users when is_online, description rendered from markdown,
  capacity bar like "12/30 confirmados", RSVP buttons (Voy / Me interesa /
  Cancelar) using shadcn <ToggleGroup>. Optimistic update on RSVP.
- /admin/eventos: shadcn data table with create/edit/delete. RHF + Zod
  form, datetime inputs, image upload for cover, markdown editor for
  description.

Deliverables: events module fully working for both roles.
